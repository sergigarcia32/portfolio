Diseño de Software

Aplicar principios de diseño en el desarrollo de software permite crear soluciones claras, coherentes y sostenibles en el tiempo. Estos principios fomentan la modularidad, la reutilización y la capacidad de adaptación ante el cambio. En conjunto, contribuyen a desarrollar software robusto, mantenible y preparado para evolucionar de manera controlada a lo largo del tiempo.

Legibilidad: con la máxima comprensibilidad para cualquier aspecto

Unicidad

Cada aspecto del sistema debe tener una única descripción o representación, evitando duplicidades o contradicciones.

Uniformidad

Se deben aplicar criterios y convenciones coherentes a lo largo de todo el diseño para mantener la consistencia.

Minimización

Cada aspecto debe tener la extensión mínima necesaria para cumplir su propósito sin agregar complejidad innecesaria.

Modularidad: con módulos separados por cada asunto, interconectados mediante sus interfaces

En la interfaz de cada módulo

Abstracción

Cada módulo debe ofrecer solo el comportamiento esencial necesario para su utilización.

Autoprotección

Los módulos deben protegerse contra un uso incorrecto o no autorizado.

Ocultación

Los detalles internos deben permanecer ocultos, mostrando solo lo necesario para su funcionamiento.

En la implementación de cada módulo

Cohesión

Cada módulo debe encargarse de un único asunto o responsabilidad, manteniendo su coherencia interna.

Desacoplamiento

Los módulos deben tener el mínimo número posible de dependencias entre sí.

Granularidad

Los módulos deben tener un tamaño reducido y bien definido, facilitando su comprensión y reutilización.

En la relación de los módulos

Jerarquía

La estructura debe organizar los módulos jerárquicamente para clarificar sus dependencias.

Dominio

El diseño debe reflejar el vocabulario y conceptos del dominio del problema.

Tecnología

La estructura debe reflejar el vocabulario y conceptos del dominio tecnológico (diseño).

Extensibilidad: con la posibilidad de añadir nuevo código minimizando la modificación del software existente

Generalización

Los módulos deben encargarse de las características comunes y generales de su grupo de colaboradores.

Especialización

Los módulos deben representar variantes o alternativas específicas de un asunto dentro de un conjunto.

Sustitución

Los módulos deben poder reemplazarse por otros equivalentes sin alterar el funcionamiento global del sistema.
